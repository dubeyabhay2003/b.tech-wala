Introduction to Java



Intro of java

Java is a versatile, object-oriented programming language designed to minimize implementation dependencies. The principle behind Java is "write once, run anywhere" (WORA), meaning that once Java code is compiled, it can run on any platform supporting Java without requiring recompilation. Introduced in 1995, Java quickly gained popularity for developing applications across desktop, web, and mobile devices. Its simplicity, robustness, and robust security features have made it a preferred choice for enterprise-level applications.

James Gosling, initially developing Java in 1991 along with Mike Sheridan and Patrick Naughton (collectively known as the Green team) at Sun Microsystems Inc., later saw Java being acquired by Oracle Corporation. Java simplifies the tasks of writing, compiling, and debugging programs, emphasizing the creation of reusable and modular code. As a class-based, object-oriented language, Java minimizes implementation dependencies and ensures that compiled Java code, in the form of bytecode, can execute on any Java Virtual Machine (JVM). The syntax of Java closely resembles that of C/C++.

Java's history unfolds with the initiation of the Java language by the Green team in 1991. The first public implementation, Java 1.0, was released by Sun Microsystems in 1996, offering no-cost run-times on various platforms. The Java 1.0 compiler underwent a significant transformation when Arthur Van Hoff rewrote it in Java, ensuring strict compliance with specifications. Subsequent versions, starting with Java 2, featured multiple configurations tailored for different platform types.

In 1997, Sun Microsystems engaged with the ISO standards body to formalize Java, but it later withdrew from the process. Despite its proprietary software status, Sun, at one point, provided many Java implementations without charge. Sun's revenue generation from Java involved selling licenses for specialized products, such as the Java Enterprise System. The open-sourcing of Java's virtual machine core code in 2006 marked a significant shift toward making Java more accessible. By 2007, Sun had completed this process, releasing all JVM core code under open-source distribution terms. Java has since continued to evolve and remains a fundamental technology in the world of software development.

On November 13, 2006, Sun Microsystems released a substantial portion of its Java Virtual Machine (JVM) as free, open-source software. This process was completed on May 8, 2007, making the core code of JVM available under open-source distribution terms.

The principles guiding the creation of Java encompassed simplicity, robustness, security, high-performance, portability, multi-threading, interpretation, dynamism, and more. In 1995, James Gosling, often referred to as the Father of Java, developed the language. Today, Java finds applications in mobile devices, internet programming, games, e-business, and various other domains.

Why is it Named Java?

Originally named OAK, the team sought a new name reflecting the essence of technology. Suggested names like Silk, Jolt, revolutionary, DNA, and dynamic were considered. Java emerged as a top choice, preferred for its uniqueness. The name has no acronym and was inspired by the island in Indonesia where the first coffee (java coffee) was produced. James Gosling chose this name during a coffee break near his office.

Java Terminology

  1. Java Virtual Machine (JVM): Known as JVM, it undergoes three execution phases: writing, compiling, and running the program. Writing is done by Java programmers, compilation is performed by the JAVAC compiler, and running involves the execution of bytecode generated by the compiler. JVM ensures platform independence.
  2. Bytecode in the Development Process: Javac compiler of JDK compiles Java source code into bytecode saved as .class files. Tools like javap can be used to view bytecode.
  3. Java Development Kit (JDK): A comprehensive development kit including a compiler, Java Runtime Environment (JRE), debuggers, and documentation. JDK installation is necessary for creating, compiling, and running Java programs.
  4. Java Runtime Environment (JRE): Included in JDK, JRE allows Java programs to run but lacks the ability to compile. It comprises a browser, JVM, applet support, and plugins.
  5. Garbage Collector: In Java, the Garbage Collector recollects objects that are no longer referenced, simplifying memory management for programmers.
  6. ClassPath: The file path where Java runtime and compiler look for .class files. JDK provides default libraries, and additional libraries must be added to the classpath.

Main Features of Java

  1. Platform Independent: Java's compiler converts source code to bytecode, allowing execution on different platforms. This architecture-neutral bytecode contributes to Java's platform-independent nature.
  2. Object-Oriented Programming Language: Java organizes programs based on objects, incorporating concepts like abstraction, encapsulation, inheritance, and polymorphism.
  3. Simple: Java eliminates complex features like pointers and explicit memory allocation, making it a straightforward language.
  4. Robust: Java emphasizes error-checking, incorporating features like garbage collection, exception handling, and efficient memory allocation.
  5. Secure: Absence of pointers in Java enhances security, preventing access to out-of-bound arrays and minimizing security flaws.
  6. Distributed: Java facilitates the creation of distributed applications using technologies like Remote Method Invocation and Enterprise Java Beans.
  7. Multithreading: Java supports concurrent execution of multiple program parts, maximizing CPU utilization.
  8. Portable: Java code written on one machine can run on another due to its platform-independent bytecode.
  9. High Performance: Java architecture, including Just In Time (JIT) compilation, enhances performance by compiling methods on demand.
  10. Dynamic Flexibility: Java's object-oriented nature provides flexibility to add classes, new methods, and create new classes through subclasses. Support for native methods in languages like C and C++ adds versatility.
  11. Sandbox Execution: Java programs run in a separate space, ensuring execution without affecting the underlying system. The bytecode verifier enhances security by checking for access violations.
  12. Write Once Run Anywhere: Java applications generate a ‘.class’ file with architecture-neutral bytecode, enabling execution on different platforms. This feature is pivotal for Java's extensive use in the IT industry.
  13. Power of Compilation and Interpretation: Java integrates both compilation and interpretation. The Java compiler converts source code to bytecode, and JVM executes this bytecode to machine OS-dependent executable code.

Java Class Terminology

  • class: Used to declare classes in Java.
  • public: An access specifier indicating visibility to all.
  • static: A keyword to make a function static. The main() method is called by JVM without creating an object for the class.
  • void: The return type, indicating the function will not return anything.
  • main: The most important method in a Java program, where all the logic must be inside. The absence of a main() method causes compilation errors.
  • String[] args: Signifies that users may enter parameters at the command line. Both forms, String[] args and String args[], are accepted by the Java compiler.
  • System.out.println: Used to print on the console, similar to “printf” in C language.

Advantages of Java:

Platform independent: Java code can run on any platform that has a Java Virtual Machine (JVM) installed, which means that applications can be written once and run on any device.

Object-Oriented: Java is an object-oriented programming language, which means that it follows the principles of encapsulation, inheritance, and polymorphism.

Security: Java has built-in security features that make it a secure platform for developing applications, such as automatic memory management and type checking.

Large community: Java has a large and active community of developers, which means that there is a lot of support available for learning and using the language.

Enterprise-level applications: Java is widely used for developing enterprise-level applications, such as web applications, e-commerce systems, and database systems.

Disadvantages of Java:

Performance: Java can be slower compared to other programming languages, such as C++, due to its use of a virtual machine and automatic memory management.

Memory management: Java’s automatic memory management can lead to slower performance and increased memory usage, which can be a drawback for some applications.

Important Points Of java:

1.Platform Independence:

Java follows the "Write Once, Run Anywhere" (WORA) principle, allowing compiled Java code to run on any device with a Java Virtual Machine (JVM).

2.Object-Oriented Programming (OOP):

Java is designed around OOP principles, promoting modularity, reusability, and flexibility in software development.

3.Security Features:

Java incorporates robust security features, including automatic memory management, type checking, and built-in security mechanisms, making it a secure platform for application development.

4.Large and Active Community:

Java has a vast and active developer community, providing extensive support, resources, and libraries for learning and implementing Java-based solutions.

5.Enterprise-Level Applications:

Widely used for building large-scale enterprise applications, Java is suitable for developing complex systems, web applications, e-commerce platforms, and database applications.

6.Portability:

Java's platform independence allows applications to be easily ported across different operating systems without modification, enhancing flexibility in deployment.

7.Versatility:

Java is versatile and used in various domains, including mobile app development (Android), web development (Java EE), scientific computing, big data processing (Hadoop), and more.

8.Multithreading Support:

Java provides built-in support for multithreading, allowing concurrent execution of multiple threads, enhancing performance in applications that require parallel processing.

9.Rich Standard Library:

Java comes with a comprehensive standard library that includes a wide range of classes and methods, simplifying common programming tasks and reducing the need for external libraries.

10.Automatic Memory Management:

The Java Virtual Machine (JVM) manages memory automatically through garbage collection, reducing the likelihood of memory-related errors and improving overall system stability.

11.Community-Driven Evolution:

Java is continuously evolving with active community involvement, receiving updates and improvements to keep up with modern programming needs.

12.Sandboxing and Security Model:

Java's sandboxing capabilities enable the execution of applications in a secure environment, protecting the host system from potential harm caused by malicious code.

13.Industry Standard for Large Systems:

Java is a preferred choice for developing large-scale, mission-critical systems in industries such as finance, healthcare, telecommunications, and more.

14.Backward Compatibility:

Java places a strong emphasis on backward compatibility, ensuring that older Java applications can run on newer versions without major modifications.

15.Community-Driven Open Source:

With a significant portion of Java being open source, the community actively contributes to its development, fostering innovation and adaptability.
