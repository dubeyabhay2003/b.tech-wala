Java Identifiers





What is Identifiers?


In Java, an identifier is a name given to a variable, method, class, package, or other program elements. Identifiers are used to uniquely identify and refer to these program elements within the code. When naming identifiers in Java, there are specific rules and conventions to follow.

1. Rules for Naming Identifiers:

  • Identifiers must begin with a letter (a to z or A to Z), currency character ($) or an underscore (_).
  • After the first character, identifiers can also contain digits (0 to 9).
  • Java is case-sensitive, so uppercase and lowercase letters are distinct. For example, "myVariable" and "MyVariable" are two different identifiers.
  • Identifiers cannot be a keyword or a reserved word in Java (e.g., int, class, if, etc.).

2. Conventions for Naming Identifiers:

  • Choose meaningful names that describe the purpose of the variable, method, class, etc.
  • Use camelCase for variable and method names (e.g., myVariable, calculateTotalAmount()).
  • Use PascalCase for class names (e.g., Person, CarModel).
  • Constants are typically written in uppercase with underscores separating words (e.g., MAX_VALUE, PI).
  • Packages are usually named in lowercase, and domain names are reversed (e.g., com.example.myproject).

Examples:

  • Valid identifiers: counter, _value, MAX_SIZE, calculateTotal, MyClass, accountBalance.
  • Invalid identifiers: 3total (starts with a digit), if (keyword), my-variable (contains a hyphen), my variable (contains a space).

Here are a few examples demonstrating the rules and conventions:


      
        // Variable names using camelCase
int myVariable;
double totalAmount;

// Method name using camelCase
void calculateTotalAmount() {
    // method body
}

// Class name using PascalCase
class MyClass {
    // class body
}

// Constant name in uppercase with underscores
final double PI = 3.14159;
static final int MAX_SIZE = 100;

// Package name in lowercase
package com.example.myproject;
